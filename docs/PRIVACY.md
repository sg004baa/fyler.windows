# 匿名フィードバックのプライバシー

fylerの`:feedback`機能は、利用者が内容を確認して「送信」を選んだ場合にだけ、
開発者へフィードバックを送信します。自動送信は行いません。

## 送信される情報

送信される情報は次の項目だけです。

- 利用者が入力した本文
- 種別（感想・要望・不具合）
- fylerのバージョン
- OSの種別
- CPUアーキテクチャ
- 送信スキーマのバージョン

本文には個人情報や公開したくない情報を入力しないでください。

## 送信されない情報

次の情報をfylerがフィードバックへ自動的に含めることはありません。

- 現在のrootパス
- ファイル名
- ファイルツリー
- bufferの内容
- bookmark
- recent root
- config
- undo journal
- ユーザー名
- 端末名
- メールアドレス
- 永続的なdevice ID
- ログ
- fyler独自のtracking identifier

## IPアドレス

IPアドレスはインターネット通信の成立に必要なため、受信基盤であるCloudflareによって
通信上処理されます。Workerは送信元IPアドレスを1分あたりの送信回数を制限する一時的な
rate limitキーとしてだけ利用します。IPアドレスをD1へ保存せず、Workerのログにも出力しません。

## 保存・閲覧・保持期間

フィードバックはCloudflare D1に保存され、fylerの開発者だけが閲覧します。
受信から12ヶ月を目安に削除します。削除を希望する場合は、
[GitHub Issues](https://github.com/sg004baa/fyler.windows/issues)から依頼してください。
フィードバックには利用者を特定するIDがないため、受信日時、種別、本文の一部など、
対象を特定できる情報を依頼に含めてください。公開Issueへ本文を書きたくない場合は、
機密情報を掲載せず、まず連絡方法を相談してください。

## 「匿名」の意味

ここでいう匿名とは、送信にアカウント、氏名、メールアドレスなどの連絡先を要求せず、
永続的なdevice IDも付与しないことを意味します。そのため、送信されたフィードバックへ
個別に返信することはできません。本文に自ら個人情報を記載した場合まで匿名性を保証するものでは
ありません。

個別返信が必要な不具合報告、再現手順、画像、追加情報を伴う報告は、
[GitHub Issues](https://github.com/sg004baa/fyler.windows/issues)を利用してください。
